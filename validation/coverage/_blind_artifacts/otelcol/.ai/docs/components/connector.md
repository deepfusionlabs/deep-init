# Component: connector

Provenance: DeepInit EXTRACT stage (BLIND, code-only re-derivation) — component `connector`, path `connector/`, otelcol (Go multi-module). Module `go.opentelemetry.io/collector/connector` (connector/go.mod:1). Every claim below cites a file:line opened inside C:/tmp/p5_otelcol_blind/connector/.

## Role
- Defines the connector abstraction: a component that acts simultaneously as an exporter from one pipeline AND a receiver to one or more downstream pipelines, joining/replicating telemetry across pipelines (connector/connector.go:16-18; the `forward` doc-comment "join pipelines together" at connector/forwardconnector/forward.go:76-78).

## Data
- No persistence / data-store owned. State is in-memory only: `BaseRouter[T]` holds `Consumers map[pipeline.ID]T` (a copy made via `maps.Copy`) and a `fanout func([]T) T` (connector/internal/router.go:16-25). No DB, file, queue, or network store is read or written in this component's source.

## Boundary rules
- A connector bridges exactly one source signal pipeline to one or more destination signal pipelines; the 9 (3×3) signal-pair Create* methods + their NxN error matrix enforce that only supported signal conversions are constructed, else `ErrDataTypes(...ErrSignalNotSupported)` (connector/connector.go:93-103, connector/connector.go:308-311, connector/internal/factory.go:13-15).
- Factory is a sealed interface: it "cannot be directly implemented" and must be built via `NewFactory`, enforced by the unexported marker method `unexportedFactoryFunc` (connector/connector.go:79-80, connector/connector.go:117, connector/connector.go:270).
- Router-and-consumer interfaces are sealed against external implementation by the unexported `privateFunc` method (connector/logs_router.go:23, connector/logs_router.go:71, connector/metrics_router.go:18, connector/traces_router.go:18).
- Stable-vs-experimental layering: the stable `connector` package handles traces/metrics/logs only; profiles support lives in the separate `xconnector` experimental wrapper that embeds `connector.Factory` and appends the stable opts (connector/xconnector/connector.go:19-20, connector/xconnector/connector.go:221, connector/xconnector/connector.go:343-348).
- `internal/` packages (router.go, factory.go) are import-restricted to this module by Go's internal-package rule (connector/internal/router.go:4, connector/internal/factory.go:4).
- Bundled `forwardconnector` is the canonical reference connector: a pass-through identity that declares `MutatesData: false` (connector/forwardconnector/forward.go:88-90).

## Key facts
- Each connector interface (Traces/Metrics/Logs) is the composition `component.Component` + `consumer.X` — that single composition is what makes one object behave as both an exporter (it consumes upstream) and a receiver (it feeds downstream consumers) (connector/connector.go:28-31, connector/connector.go:44-47, connector/connector.go:59-62).
- The factory is option-built (functional-options pattern): `NewFactory(type, defaultCfg,...FactoryOption)` applies `WithTracesToTraces`-style options that each install a Create* func + a `component.StabilityLevel`; unset signal-pairs default to returning `ErrDataTypes` (connector/connector.go:163-167, connector/connector.go:309-311, connector/connector.go:417-427).
- `BaseRouter[T any]` is a Go generic shared across all four signal routers (logs/metrics/traces/profiles), parameterizing `Consumer(...pipeline.ID)` and `PipelineIDs` over the consumer type (connector/internal/router.go:16, connector/logs_router.go:28, connector/metrics_router.go:23, connector/xconnector/profiles_router.go:22).
- A router feeds the FIRST consumer of each named pipeline and fans out across all pipelines; `Consumer(ids...)` builds a fan-out over the selected pipelines and returns an aggregated error (via `go.uber.org/multierr`) if any pipeline ID is missing (connector/logs_router.go:18, connector/logs_router.go:50-68, connector/internal/router.go:35-54, connector/go.mod:16).
- `Settings` uses the unkeyed-literal guard idiom — a trailing `_ struct{}` field forces keyed construction (connector/connector.go:73-74).
- Manifest classifies this as a pkg-class connector component for project open-telemetry/opentelemetry-collector (connector/metadata.yaml:1-5).
- Go 1.25.0 module; uses generics + the std `maps` package (connector/go.mod:3, connector/internal/router.go:9).
- The `forward` reference connector embeds all four consumer interfaces plus `xconsumer.Profiles` and the `component.StartFunc`/`ShutdownFunc` no-op lifecycle helpers, making one struct serve every signal-to-same-signal forward (connector/forwardconnector/forward.go:79-86).
