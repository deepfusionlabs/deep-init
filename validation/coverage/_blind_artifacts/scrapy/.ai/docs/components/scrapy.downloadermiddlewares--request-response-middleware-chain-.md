# scrapy.downloadermiddlewares (request/response middleware chain)

A package of independently-loadable downloader middleware classes, each hooking the per-request download path with `process_request` / `process_response` / `process_exception`. The package itself has no `__init__` code (`scrapy/downloadermiddlewares/__init__.py:1` is empty, 1 line) — each module is a standalone component.

## Role

- A collection of pluggable downloader-middleware classes (cookies, redirect, retry, robots.txt, http-compression, http-proxy, http-cache, http-auth, user-agent, offsite, stats, default-headers, download-timeout) that observe and transform requests on their way to the downloader and responses on their way back, each implementing the `process_request`/`process_response`/`process_exception` hook contract — e.g. `scrapy/downloadermiddlewares/cookies.py:40` `class CookiesMiddleware`, `scrapy/downloadermiddlewares/redirect.py:198` `class RedirectMiddleware`, `scrapy/downloadermiddlewares/retry.py:141` `class RetryMiddleware`.

## Data

- **No durable store owned by the package directly.** State is in-memory per-middleware instance:
 - cookie jars keyed by `cookiejar` meta — `self.jars: defaultdict[Any, CookieJar]` (`scrapy/downloadermiddlewares/cookies.py:46`).
 - robots.txt parsers cached per netloc, value may be an in-flight `Deferred` — `self._parsers: dict[str, RobotParser | Deferred | None]` (`scrapy/downloadermiddlewares/robotstxt.py:43`).
 - seen offsite domains — `self.domains_seen: set[str]` (`scrapy/downloadermiddlewares/offsite.py:29`).
 - proxy table from environment — `self.proxies` populated from `getproxies` (`scrapy/downloadermiddlewares/httpproxy.py:29-36`).
- **HTTP cache persistence is delegated, not owned** — `HttpCacheMiddleware` loads a pluggable `HTTPCACHE_STORAGE` + `HTTPCACHE_POLICY` via `load_object` and calls `storage.retrieve_response` / `storage.store_response` (`scrapy/downloadermiddlewares/httpcache.py:47-48`,`:80`,`:149`); the actual on-disk store lives behind that setting, outside this package.
- **Stats collector (cross-component telemetry sink)** — many middlewares write counters via `crawler.stats.inc_value(...)`, e.g. `retry/count` (`scrapy/downloadermiddlewares/retry.py:121`), `httpcache/hit` (`scrapy/downloadermiddlewares/httpcache.py:93`), `robotstxt/forbidden` (`scrapy/downloadermiddlewares/robotstxt.py:81`), `downloader/response_bytes` (`scrapy/downloadermiddlewares/stats.py:72`), `offsite/filtered` (`scrapy/downloadermiddlewares/offsite.py:64`).
- **Request.meta is the cross-hook scratch store** — flags/counters passed between hooks and between retries: `dont_merge_cookies`/`cookiejar` (`scrapy/downloadermiddlewares/cookies.py:79`,`:82`), `redirect_times`/`redirect_ttl`/`redirect_urls` (`scrapy/downloadermiddlewares/redirect.py:94-107`), `retry_times` (`scrapy/downloadermiddlewares/retry.py:98`,`:110`), `_dont_cache`/`cached_response` (`scrapy/downloadermiddlewares/httpcache.py:76`,`:98`), `proxy`/`_auth_proxy`/`_scheme_proxy` (`scrapy/downloadermiddlewares/httpproxy.py:89-96`).

## Boundary rules

- **Hook-contract layering** — each class exposes only `process_request` (request→down), `process_response` (response→up), and/or `process_exception`; the manager builds three method lists, with response/exception lists reversed (`appendleft`) vs request order (`scrapy/core/downloader/middleware.py:45-52`). A hook may return `None` (continue), a `Response`/`Request` (short-circuit), or raise (`scrapy/core/downloader/middleware.py:107-114`).
- **Priority-ordered chain (not import order)** — placement is governed by integer priorities in `DOWNLOADER_MIDDLEWARES_BASE`, e.g. Offsite=50, RobotsTxt=100, HttpAuth=300, UserAgent=500, Retry=550, HttpCompression=590, Redirect=600, Cookies=700, HttpProxy=750, DownloaderStats=850, HttpCache=900 (`scrapy/settings/default_settings.py:317-330`).
- **`from_crawler` is the only construction boundary** — middlewares read settings/stats/signals exclusively through the `Crawler` handed to `from_crawler`, raising `NotConfigured` to opt out when their enable-setting is off (`scrapy/downloadermiddlewares/cookies.py:51-52`, `scrapy/downloadermiddlewares/retry.py:145-146`, `scrapy/downloadermiddlewares/httpcompression.py:84-85`).
- **`request.meta` opt-out convention** — callers bypass a middleware per-request via meta flags the middleware checks first: `dont_redirect`/`handle_httpstatus_*` (`scrapy/downloadermiddlewares/redirect.py:208-214`), `dont_retry` (`scrapy/downloadermiddlewares/retry.py:169`), `dont_obey_robotstxt` (`scrapy/downloadermiddlewares/robotstxt.py:59`), `dont_cache` (`scrapy/downloadermiddlewares/httpcache.py:71`), `dont_merge_cookies` (`scrapy/downloadermiddlewares/cookies.py:79`), `dont_filter`/`allow_offsite` (`scrapy/downloadermiddlewares/offsite.py:50-51`).
- **Cross-origin header-stripping is a security boundary** — on redirect, `Cookie` is dropped unless same-host + scheme not downgraded, and `Authorization` is dropped on any scheme/host/port change (CORS non-wildcard rule) (`scrapy/downloadermiddlewares/redirect.py:162-174`); proxy creds are stripped on a scheme change (`scrapy/downloadermiddlewares/redirect.py:132-139`).

## Key facts

- **Middleware-vs-runner split** — this package holds the *policies*; the *engine* that orders and drives them (`DownloaderMiddlewareManager`) lives in a separate component, `scrapy.core.downloader` (`scrapy/core/downloader/middleware.py:34`). Discovery is duck-typed by `hasattr` on each hook name (`scrapy/core/downloader/middleware.py:44-50`).
- **`redirect.py` reaches across the chain into the spider-middleware side** — `BaseRedirectMiddleware` borrows the live `RefererMiddleware` instance (a *spider* middleware) to compute redirect Referer headers, warning at `engine_started` if none is enabled (`scrapy/downloadermiddlewares/redirect.py:66-91`). This is the one downloader→spider-middleware coupling.
- **RobotsTxtMiddleware is re-entrant via the engine** — it triggers an *additional* download (`crawler.engine.download_async`) for `/robots.txt`, marking that sub-request `dont_obey_robotstxt` to avoid recursion, and latches concurrent same-netloc requests on a single `Deferred` (`scrapy/downloadermiddlewares/robotstxt.py:88-116`); its hook is `async def` (`scrapy/downloadermiddlewares/robotstxt.py:56`).
- **Cookie domain check uses the public-suffix list** — `_is_public_domain` via `TLDExtract(include_psl_private_domains=True)` rejects setting cookies on bare public suffixes unless the host matches exactly (`scrapy/downloadermiddlewares/cookies.py:31-37`,`:68-71`).
- **Decompression is bounded + content-negotiated** — `ACCEPTED_ENCODINGS` starts `[gzip, deflate]` and conditionally appends `br`/`zstd` based on installed libs (`scrapy/downloadermiddlewares/httpcompression.py:32-59`); decompression enforces `DOWNLOAD_MAXSIZE`, raising `IgnoreRequest` on `_DecompressionMaxSizeExceeded` (a decompression-bomb guard) (`scrapy/downloadermiddlewares/httpcompression.py:120-126`).
- **Retry logic is exposed as a reusable free function** — `get_retry_request(...)` is module-level and documented for use directly in spider callbacks, separate from `RetryMiddleware` which merely wraps it for the response/exception hooks (`scrapy/downloadermiddlewares/retry.py:38`,`:198`).
- **HttpCache acts as an error-recovery fallback** — `process_exception` can serve a stale `cached_response` when a download fails with one of `DOWNLOAD_EXCEPTIONS`, incrementing `httpcache/errorrecovery` (`scrapy/downloadermiddlewares/httpcache.py:134-143`).
- **Twisted is the async substrate** — robots.txt concurrency uses raw `twisted.internet.defer.Deferred` and `maybe_deferred_to_future` (`scrapy/downloadermiddlewares/robotstxt.py:12`,`:18`,`:89`,`:115`); the manager runs every hook through `ensure_awaitable`, so hooks may be sync or `async` (`scrapy/core/downloader/middleware.py:103`).
- **Every hook carries `_warn_spider_arg`** — the legacy `spider` positional arg is deprecation-shimmed at the decorator layer (`scrapy/utils/decorators` import `scrapy/downloadermiddlewares/cookies.py:12`; applied at `scrapy/downloadermiddlewares/cookies.py:75`,`:93` and on every other hook), and the manager injects `spider` only for methods that still require it (`scrapy/core/downloader/middleware.py:90-94`).
